Basic SQL Statements — الأوامر الأساسية
Command Purpose بالعربي
SELECT Retrieve data from one or more tables استرجاع البيانات
WHERE Filter rows based on a condition تصفية الصفوف بشرط
ORDER BY Sort results ASC or DESC ترتيب النتائج تصاعدياً أو تنازلياً
GROUP BY Group rows sharing a value for aggregation تجميع الصفوف لحسابات إجمالية
HAVING Filter groups (used after GROUP BY, like WHERE for groups) تصفية المجموعات بعد GROUP BY
INSERT Add new rows to a table إضافة سجلات جديدة
UPDATE Modify existing rows تعديل سجلات موجودة
DELETE Remove rows from a table حذف سجلات
SQL Examples
SELECT Name, Salary FROM Employees WHERE Salary > 5000 ORDER BY Salary DESC; SELECT Dept, COUNT(*) AS Total FROM Employees GROUP BY Dept HAVING COUNT(*) > 3; INSERT INTO Employees (Name, Salary) VALUES ('Ali', 7000); UPDATE Employees SET Salary = 8000 WHERE ID = 5; DELETE FROM Employees WHERE ID = 5;
Arabic
SELECT: جلب البيانات من الجدول. WHERE: تصفية النتائج بشرط معين. ORDER BY: ترتيب النتائج. GROUP BY: تجميع الصفوف لعمليات مثل COUNT وSUM. HAVING: شرط بعد التجميع (مثل WHERE لكن للمجموعات). INSERT: إضافة سجل. UPDATE: تعديل سجل. DELETE: حذف سجل.
SQL JOINs — الربط بين الجداول

JOINs combine rows from two or more tables based on a related column.

JOIN Type Returns بالعربي
INNER JOIN Only matching rows in BOTH tables الصفوف المتطابقة فقط في الجدولين
LEFT JOIN ALL rows from left table + matching from right (NULL if no match) كل صفوف الجدول الأيسر + المطابق من الأيمن
RIGHT JOIN ALL rows from right table + matching from left كل صفوف الجدول الأيمن + المطابق من الأيسر
FULL JOIN All rows from both tables كل صفوف الجدولين
-- INNER JOIN: only customers who placed orders SELECT C.Name, O.OrderDate FROM Customers C INNER JOIN Orders O ON C.ID = O.CustomerID; -- LEFT JOIN: all customers, with order if exists SELECT C.Name, O.OrderDate FROM Customers C LEFT JOIN Orders O ON C.ID = O.CustomerID;
Arabic
INNER JOIN: يجلب فقط الصفوف التي لها تطابق في كلا الجدولين. LEFT JOIN: يجلب كل صفوف الجدول الأيسر حتى لو لم يكن لها تطابق في الأيمن (يظهر NULL). RIGHT JOIN: عكس LEFT JOIN. القاعدة: دائماً JOIN يحتاج ON لتحديد عمود الربط.
Primary Key & Foreign Key المفاتيح

Primary Key (PK): Uniquely identifies each row in a table. Cannot be NULL. Only one per table.

Foreign Key (FK): A column that references the Primary Key of another table. Creates a relationship between tables.

CREATE TABLE Orders ( OrderID INT PRIMARY KEY, CustID INT FOREIGN KEY REFERENCES Customers(ID) );
Arabic
Primary Key: مفتاح أساسي يحدد كل سجل بشكل فريد — لا يتكرر ولا يكون فارغاً. Foreign Key: مفتاح أجنبي يربط جدولين معاً عن طريق الإشارة لـ PK جدول آخر.
Normalization — التطبيع

Normalization organizes a database to reduce redundancy and improve integrity.

Form Rule
1NF No repeating groups; each cell has one value; has PK
2NF 1NF + no partial dependency on composite PK
3NF 2NF + no transitive dependency (non-key → non-key)
Arabic
التطبيع هو تنظيم الجداول لتقليل التكرار. 1NF: لا تكرار وكل خلية قيمة واحدة. 2NF: لا اعتماد جزئي على المفتاح. 3NF: لا اعتماد عبوري (قيمة تعتمد على قيمة غير مفتاح).
Stored Procedures — الإجراءات المخزنة

A Stored Procedure is a precompiled set of SQL statements stored in the database. Faster than raw queries, reusable, and can accept parameters.

CREATE PROCEDURE GetEmployee @EmpID INT AS BEGIN SELECT * FROM Employees WHERE ID = @EmpID; END; -- Execute the procedure: EXEC GetEmployee @EmpID = 5;
Arabic
الـ Stored Procedure هو مجموعة أوامر SQL محفوظة في قاعدة البيانات وتُنفَّذ بأمر واحد. مميزاتها: أسرع (precompiled)، قابلة لإعادة الاستخدام، وتقبل معاملات (Parameters). تُستخدم كثيراً للعمليات المتكررة مثل استرجاع بيانات موظف بـ ID معين.

IP Address & DNS العنونة وترجمة الأسماء

IP Address: 32-bit (IPv4) unique identifier for a device on a network. Written as 4 octets: 192.168.1.1

DNS (Domain Name System): Translates human-readable domain names (google.com) into IP addresses (142.250.185.78). Like a phone book for the internet.

Arabic
IP Address: عنوان فريد لكل جهاز على الشبكة. مكون من 4 أرقام (0-255) مفصولة بنقاط. DNS: نظام يترجم اسم الموقع (مثل google.com) إلى عنوان IP — مثل دليل الهاتف للإنترنت.
HTTP vs HTTPS & LAN/WAN البروتوكولات والشبكات

HTTP: Transfers web pages. Data sent in plain text — not secure.

HTTPS: HTTP + TLS encryption. Data is encrypted — secure. Uses port 443.

LAN: Local Area Network — limited area (home, office). Fast, private.

WAN: Wide Area Network — large geographic area (Internet is the biggest WAN).

Arabic
HTTP: بيانات مكشوفة (غير آمن). HTTPS: مشفر بـ TLS (آمن، المنفذ 443). LAN: شبكة محلية (منزل، مكتب). WAN: شبكة واسعة تمتد لمسافات كبيرة كالإنترنت.
Router vs Switch — الفرق بين الأجهزة الشبكية
Device OSI Layer Function Addresses بالعربي
Hub Layer 1 Broadcasts all data to all ports None يرسل البيانات لكل الأجهزة
Switch Layer 2 Sends frames to the correct device only using MAC MAC Address يوجه البيانات للجهاز الصحيح بالـ MAC
Router Layer 3 Routes packets between different networks using IP IP Address يوجه الحزم بين الشبكات المختلفة بالـ IP
Arabic
Hub: يرسل البيانات لكل الأجهزة بدون تمييز (قديم). Switch: أذكى من الـ Hub، يعرف MAC كل جهاز ويرسل البيانات للجهاز الصحيح فقط. Router: يربط شبكات مختلفة ببعض ويعرف أفضل مسار لإرسال البيانات عبر الإنترنت.

Authentication vs Authorization التحقق والصلاحية

Authentication: Verifying who you are. (Username + Password, fingerprint, OTP)

Authorization: Determining what you can do after authentication. (Admin can delete, user can only read)

🔑 Authentication = Identity   |   Authorization = Permissions
Arabic
Authentication (التحقق): من أنت؟ مثل إدخال كلمة المرور. Authorization (التفويض): ماذا يحق لك أن تفعل؟ مثلاً المدير يستطيع الحذف، المستخدم العادي يقرأ فقط. التحقق يأتي أولاً دائماً.
Encryption vs Hashing التشفير والتجزئة

Encryption: Converts data to unreadable form using a key. Reversible — can decrypt with the right key. (AES, RSA)

Hashing: Converts data to a fixed-size digest. One-way — cannot reverse. Used for passwords. (MD5, SHA-256, bcrypt)

Arabic
Encryption (التشفير): يحول البيانات لشكل مشفر يمكن فك تشفيره بالمفتاح. Hashing (التجزئة): يحول البيانات لرمز ثابت لا يمكن عكسه. نستخدمه لتخزين كلمات المرور — حتى لو سُرقت قاعدة البيانات، كلمات المرور غير مكشوفة.
Common Attacks — الهجمات الشائعة
Attack What It Does Prevention بالعربي
SQL Injection Attacker injects malicious SQL into input fields to manipulate the database Use parameterized queries / prepared statements حقن كود SQL خبيث عبر حقول الإدخال
XSS Cross-Site Scripting: inject malicious JavaScript into web pages viewed by users Sanitize/escape all user input; Content Security Policy حقن JavaScript خبيث في صفحات الموقع
Phishing Fake emails/sites trick users into revealing credentials User education; email filters; MFA مواقع وإيميلات مزيفة لسرقة بيانات المستخدم
CSRF Forces authenticated user to perform unwanted actions CSRF tokens; SameSite cookies إجبار المستخدم على تنفيذ أوامر غير مقصودة
SQL Injection Example
-- Vulnerable query (NEVER do this): "SELECT * FROM users WHERE name = '" + userInput + "'" -- If userInput = "'; DROP TABLE users; --" → DISASTER! -- Safe: Parameterized Query SELECT * FROM users WHERE name = @name -- value passed separately
Arabic
SQL Injection: أخطر هجمة على قواعد البيانات. يدخل المهاجم كود SQL في حقل نصي ليتحكم بقاعدة البيانات. الحل: استخدام Parameterized Queries دائماً وليس بناء استعلام بدمج نص المستخدم مباشرة. XSS: يحقن JavaScript في الصفحة ليسرق جلسات المستخدمين. Phishing: تصيد احتيالي بمواقع مزيفة.
Firewall & Malware جدار الحماية والبرمجيات الخبيثة

Firewall: Monitors and controls network traffic based on rules. Blocks unauthorized access. Can be hardware or software.

Malware: Malicious software. Types: Virus Worm Trojan Ransomware Spyware

Arabic
Firewall: جدار ناري يراقب حركة الشبكة ويمنع الوصول غير المصرح به. Malware: برمجيات خبيثة — Virus (يلصق بملفات)، Worm (ينتشر وحده)، Trojan (يتنكر كبرنامج مفيد)، Ransomware (يشفر ملفاتك ويطلب فدية).

GET vs POST — Sessions & Cookies إرسال البيانات والجلسات
Feature GET POST
Data location URL query string (?name=ali) Request body (hidden)
Security Less secure (visible) More secure
Data limit ~2000 chars No practical limit
Use case Search, filters, bookmarkable URLs Login, forms, file upload
// GET: data in URL → ?id=5 $id = $_GET['id']; // POST: data in body $user = $_POST['username']; // Sessions: server-side storage across pages session_start(); $_SESSION['user'] = "Ali"; // Cookies: client-side storage setcookie("theme", "dark", time()+86400); echo $_COOKIE['theme'];
Arabic
GET: البيانات في الـ URL — مناسب للبحث، غير مناسب لكلمات المرور. POST: البيانات مخفية في الطلب — للفورمات الحساسة. Session: تخزن البيانات على السيرفر بين الصفحات (تنتهي عند إغلاق المتصفح). Cookie: تخزن على جهاز المستخدم ويمكن تحديد وقت انتهائها.
PHP Database Connection & Basic CRUD الاتصال بقاعدة البيانات
// Connect to MySQL database $conn = new mysqli("localhost", "root", "", "mydb"); if ($conn->connect_error) die("Connection failed"); // READ (Select) $result = $conn->query("SELECT * FROM users"); while ($row = $result->fetch_assoc()) { echo $row['name']; } // CREATE (Insert) — safe with prepared statement $stmt = $conn->prepare("INSERT INTO users(name) VALUES(?)"); $stmt->bind_param("s", $name); $stmt->execute();
Arabic
للاتصال بقاعدة البيانات نستخدم mysqli أو PDO. CRUD = Create (INSERT)، Read (SELECT)، Update (UPDATE)، Delete (DELETE). دائماً استخدم Prepared Statements لمنع SQL Injection.

OOP Principles — مبادئ البرمجة كائنية التوجه
1. Encapsulation — التغليف
Hide internal data; expose only what's needed via public methods/properties. Protects object state.
Use private fields with public getters/setters.
2. Inheritance — الوراثة
A child class inherits properties and methods from a parent class. Promotes code reuse.
class Manager : Employee { public string Department; }
3. Polymorphism — تعدد الأشكال
Same method name, different behavior depending on the object type. Achieved via method overriding and overloading.
virtual + override keywords in C#.
4. Abstraction — التجريد
Hide complex implementation; show only essential features. Achieved with abstract classes and interfaces.
User uses Start() without knowing HOW it starts.
Arabic
Encapsulation (التغليف): إخفاء التفاصيل الداخلية وإظهار ما يحتاجه المستخدم فقط — مثل ريموت التلفزيون: تضغط زر دون معرفة الدوائر الداخلية.
Inheritance (الوراثة): الكلاس الابن يرث خصائص ودوال الكلاس الأب — مثلاً Manager يرث من Employee.
Polymorphism (تعدد الأشكال): نفس اسم الدالة لكن سلوك مختلف حسب الكائن. مثلاً دالة Speak() تعمل بشكل مختلف في Dog وCat رغم اشتراكهما في Animal.
Abstraction (التجريد): إخفاء التعقيد وإظهار الجوهر فقط — مثل قيادة السيارة: تستخدم المقود دون معرفة كيفية عمل المحرك.

Scrum Framework إطار عمل Scrum

Scrum is an Agile framework with defined roles and ceremonies:

  • Sprint: 2-4 week development cycle
  • Product Owner: defines requirements (backlog)
  • Scrum Master: facilitates the team
  • Dev Team: builds the product
  • Daily Standup: 15-min daily meeting
  • Sprint Review: demo at end of sprint
  • Retrospective: improve next sprint
Arabic
Scrum هو أشهر إطار عمل Agile. العمل يُقسم لـ Sprints (دورات 2-4 أسابيع). الفريق يلتقي يومياً 15 دقيقة (Standup). في نهاية كل Sprint يعرضون ما أنجزوه للعميل.

Logical Reasoning — التفكير المنطقي

Pattern Recognition: Find the next number or shape in a sequence.

Example
2, 4, 8, 16, __  →  32 (multiply by 2 each time)
1, 4, 9, 16, 25, __  →  36 (perfect squares: 6²)
A is taller than B. B is taller than C. Is A taller than C?  →  Yes (transitive relation)

Key Reasoning Types:

  • Deductive: From general to specific (All humans die → Socrates is human → Socrates will die)
  • Inductive: From specific to general (pattern from examples)
  • Analogical: A:B :: C:D (Doctor:Hospital :: Teacher:?School)

Top Exam Questions — أبرز أسئلة الامتحان
Q1
What is the difference between INNER JOIN and LEFT JOIN?
Answer: INNER JOIN returns only rows with matching values in BOTH tables. LEFT JOIN returns ALL rows from the left table and matching rows from the right table (NULL if no match).
Arabic
INNER JOIN: يجلب فقط الصفوف التي لها تطابق في كلا الجدولين. LEFT JOIN: يجلب كل صفوف الجدول الأيسر بغض النظر عن التطابق، والصفوف غير المتطابقة تظهر NULL من الجدول الأيمن.
Q2
What are the 4 OOP principles? Briefly explain each.
Answer: Encapsulation (hide internal data), Inheritance (child inherits from parent), Polymorphism (same method, different behavior), Abstraction (hide complexity, show essentials).
Arabic
التغليف: إخفاء البيانات. الوراثة: الابن يرث من الأب. تعدد الأشكال: نفس الاسم سلوك مختلف. التجريد: إخفاء التعقيد وإظهار الجوهر.
Q3
What is SQL Injection and how do you prevent it?
Answer: SQL Injection is when an attacker inserts malicious SQL code into input fields to manipulate the database. Prevention: Always use parameterized queries / prepared statements — NEVER concatenate user input directly into SQL strings.
Arabic
حقن SQL: المهاجم يدخل كود SQL خبيث في حقول الإدخال. الحل: استخدام Parameterized Queries دائماً ولا تدمج نص المستخدم مباشرة في الاستعلام.
Q4
Explain the difference between Authentication and Authorization with examples.
Answer: Authentication verifies your identity (who are you? → login with username/password). Authorization determines your permissions (what can you do? → admin can delete, user can only read).
Arabic
التحقق (Authentication): من أنت؟ مثل تسجيل الدخول بالإيميل وكلمة المرور. التفويض (Authorization): ماذا يحق لك؟ مثل أن المدير يستطيع حذف المستخدمين لكن الموظف العادي لا يستطيع.
Q5
What is the OSI model? Name all 7 layers in order.
Answer: The OSI model divides network communication into 7 layers: (1) Physical, (2) Data Link, (3) Network, (4) Transport, (5) Session, (6) Presentation, (7) Application. Mnemonic: "Please Do Not Throw Sausage Pizza Away"
Arabic
نموذج OSI يقسم الاتصال لـ 7 طبقات: 1-Physical (بتات)، 2-DataLink (إطارات+MAC)، 3-Network (حزم+IP)، 4-Transport (TCP/UDP)، 5-Session (جلسات)، 6-Presentation (تشفير+ضغط)، 7-Application (تطبيقات المستخدم).
